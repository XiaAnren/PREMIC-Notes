## 20260527-开放SWAN数据权限

### 问题描述

`premdev`在2026-05-25把`/data1/premdev/datainput_arc/radar/raw/`下的SWAN数据的权限都给关上了，但是其他用户还有使用需求，所以考虑在不修改源文件权限的前提下进行操作。

### 被否决的方案

- 每次需要使用数据的时候提交申请，由管理员将对应数据复制一份出来：麻烦，而且占用空间。
- Bind mount：仍然需要修改源文件权限。

### 总体需求

- 在不修改源文件权限的前提下为其他用户开放使用权限。
- 只为`premic`组用户开启只读权限。

### 解决方案

使用`bindfs`对源文件进行镜像，同时修改目标挂载点处所属与权限（但挂载的时候`df`、`mount`、`cat /proc/mounts`都能看到，所以考虑使用完及时卸载；此外尝试`fusermount -u /data3/XiaAnRen/SWAN`卸载会报错，因此排除该方案）：
```bash
>>> sudo /public/home/XiaAnRen/software/bindfs/bin/bindfs --mirror-only=@premic --perms=g+rD -o ro /data1/premdev/datainput_arc/radar/raw /data3/XiaAnRen/SWAN
>>> sudo umount /data3/XiaAnRen/SWAN
```

<details>
<summary>安装过程</summary>

安装`libfuse`依赖（安装前缀不能是源码目录本身，否则`make install`会试图将文件复制到自身，从而产生冲突）：
```bash
>>> mkdir -p $HOME/software/fuse && cd $HOME/software/fuse
>>> wget https://github.com/libfuse/libfuse/releases/download/fuse-2.9.9/fuse-2.9.9.tar.gz
>>> tar -xzvf fuse-2.9.9.tar.gz

>>> cd $HOME/software/fuse/fuse-2.9.9
>>> ./configure --prefix=$HOME/software/fuse
>>> make
>>> make install
```

补全`pkg-config`（否则编译`bindfs`过程中会提示`configure: error: FUSE not found`）：
```bash
>>> mkdir -p $HOME/software/fuse/lib/pkgconfig && cd $HOME/software/fuse/lib/pkgconfig
>>> ln -s ../../fuse-2.9.9/fuse.pc
```

验证安装完成：
```bash
>>> PKG_CONFIG_PATH=$HOME/software/fuse/lib/pkgconfig pkg-config --modversion fuse
    2.9.9
```

安装`bindfs`：
```bash
>>> wget https://bindfs.org/downloads/bindfs-1.18.4.tar.gz
>>> tar -xzvf bindfs-1.18.4.tar.gz
>>> mv bindfs-1.18.4 $HOME/software/bindfs

>>> export CFLAGS="-I$HOME/software/fuse/include"
>>> export LDFLAGS="-L$HOME/software/fuse/lib"
>>> export PKG_CONFIG_PATH="$HOME/software/fuse/lib/pkgconfig"

>>> cd $HOME/software/bindfs
>>> ./configure --prefix=$HOME/software/bindfs
>>> make
>>> make install
```

验证安装完成：
```bash
>>> /public/home/XiaAnRen/software/bindfs/bin/bindfs --version
    bindfs 1.18.4
```
