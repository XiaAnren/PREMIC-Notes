## 20260318-SWAN数据后处理

预先对目标文件的格式进行了检查：
```bash
>>> ncdump -h "$infile"
    netcdf \\"$infile" {
    dimensions:
      Time = UNLIMITED ; // (1 currently)
      DateStrLen = 19 ;
      vertical = 21 ;
      south_north = 215 ;
      west_east = 270 ;
    variables:
      char Times(Time, DateStrLen) ;
        Times:name = "date" ;
        Times:_FillValue = "" ;
      int Level(Time, vertical) ;
        Level:_FillValue = -999 ;
      int dz(Time, vertical, south_north, west_east) ;
        dz:ordering = "XY  " ;
        dz:_FillValue = -999 ;

    // global attributes:
        :creation_date = "DDD MMM dd HH:MM:SS UTC YYYY" ;
    }
```
知道文件内的变量名以及对应缺省值是$-999$。

### 将$\mathrm{Level} \leqslant 3000 \, \mathrm{m}$对应的$\mathrm{dz}$设为缺省值

对于旧版本的`NCO`：
```bash
>>> ncap2 --version
    NCO netCDF Operators version 4.7.7 built by root on mgt at Jul  1 2019 11:52:01
    ncap2 version 4.7.7
```
`ncap2`的脚本语法解析器对属性的直接引用支持不够完善，或者在`where`语句中无法直接解析`var.attr`这种写法，导致将`var.attr`识别为了一个函数调用或其他非法token：
```bash
>>> ncap2 -s 'where(Level <= 3000) dz=dz._FillValue;' "$infile" "$outfile"
    Command-line script:1:28: expecting FUNC, found '_FillValue'
```

因此使用`ncap2`的条件赋值功能时指定具体的数值（$-999$）：
```bash
>>> ncap2 -s 'where(Level <= 3000) dz=-999;' "$infile" "$outfile"
```

### 将$\mathrm{dz}$整体增加$5 \, \mathrm{dBZ}$

使用`ncap2`的算术运算功能并指定具体的数值（$-999$）：
```bash
>>> ncap2 -s 'where(dz != -999) dz=dz+5;' "$infile" "$outfile"
```

结果发现输入输出文件内的值完全一样。推测是当前版本的`ncap2`的脚本引擎处理`where`语句中的原地修改有时不够健壮，或者因为输入文件变量定义问题，导致它读取了数据，计算了结果，但没有成功写回文件。

因此考虑创建中间变量作为跳板进行改值并删除，但当前的`NCO`版本似乎不支持`delete()`函数：
```bash
>>> ncap2 -s 'dbz=dz; where(dz!=-999) dbz=dz+5; dz=dbz; delete(dbz);' "$infile" "$outfile"
    ncap2: WARNING prs_cls::ncap_var_init(): Unable to find variable delete in "$infile" or "$outfile"
    Segmentation fault
```

明确中间变量不会对后续处理产生影响所以选择将中间变量保留：
```bash
>>> ncap2 -s 'dbz=dz; where(dz!=-999) dbz=dz+5; dz=dbz;' "$infile" "$outfile"
```

处理完成后可以使用`ncdiff`进行检查：
```bash
>>> ncdiff "$outfile" "$infile" difference.nc
```

### 将$\mathrm{dz}$随机增加$0 \text{--} 5 \, \mathrm{dBZ}$

当前版本的`NCO`无法实现随机加值的需求：
```bash
>>> ncap2 -s 'dbz=dz; where(dz!=-999) dbz=int(dz + (5.0 * rnd())); dz=dbz;' "$infile" "$outfile"
    ncap2: WARNING prs_cls::ncap_var_init(): Unable to find variable rnd in "$infile" or "$outfile"
    Segmentation fault
```

并且`ncap2`的随机数函数`rnd()`通常只能生成一个单一标量，而不是一个与变量维度相匹配的数组。意味着如果尝试写`dz + rnd()`，只会给`dz`的每一个格点都加上同一个随机数，而不是每个格点加不同的随机数。
