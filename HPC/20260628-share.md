## 20260628-共享服务器数据

### 问题描述

`Himawari8`的数据在`admin`与`mgt`上分别存储，使用起来非常不方便，因此考虑共享两个服务器的相关数据。

### 被否决的方案

- NFSv4 over SSH：性能优秀但是需要使用管理员权限修改一堆配置并持久化挂载。

### 解决方案

使用`sshfs`通过普通的SSH协议进行挂载：
```bash
>>> # mgt
>>> sudo /public/home/XiaAnRen/software/sshfs/bin/sshfs admin:/data /data3/XiaAnRen/admin-data \
>>>      -F /public/home/XiaAnRen/.ssh/config -o IdentityFile=/public/home/XiaAnRen/.ssh/id_rsa -o ro,allow_other,reconnect
>>> sudo fusermount -u /data3/XiaAnRen/admin-data
>>> # admin
>>> sudo /public/home/XiaAnRen/software/sshfs/bin/sshfs mgt:/data4 /data/mgt-data4 \
>>>      -F /public/home/XiaAnRen/.ssh/config -o IdentityFile=/public/home/XiaAnRen/.ssh/id_rsa -o ro,allow_other,reconnect
>>> sudo fusermount -u /data/mgt-data4
```

<details>
<summary>安装过程</summary>

安装`sshfs`（`pkgconfig`文件已经指定了`include`和`lib`，所以不需要额外设置`CFLAGS`和`LDFLAGS`，只需要使用`-Wl,-rpath`规定运行时的动态链接库路径）：
```bash
>>> mkdir -p $HOME/software/sshfs && cd $HOME/software/sshfs
>>> wget https://github.com/libfuse/sshfs/releases/download/sshfs-2.10/sshfs-2.10.tar.gz
>>> tar -xzvf sshfs-2.10.tar.gz

>>> export LDFLAGS="-Wl,-rpath,$HOME/software/fuse/lib -Wl,-rpath,$CONDA_PREFIX/lib"
>>> export PKG_CONFIG_PATH="$HOME/software/fuse/lib/pkgconfig:$CONDA_PREFIX/lib/pkgconfig"

>>> cd $HOME/software/sshfs/sshfs-2.10
>>> ./configure --prefix=$HOME/software/sshfs
>>> make
>>> make install
```

验证安装完成：
```bash
>>> /public/home/XiaAnRen/software/sshfs/bin/sshfs --version
    SSHFS version 2.10
    FUSE library version: 2.9.9
    fusermount version: 2.9.9
    using FUSE kernel interface version 7.19
```

</details>
