## 20260115-重复挂载

### 问题描述

`root`以外的普通用户无法正常登录，始终卡死在`ssh node15`这里。

### 问题排查

输出详细信息发现卡在认证阶段：
```bash
>>> ssh -v node15
    ...
    debug1: Authentications that can continue: publickey,gssapi-keyex,gssapi-with-mic,password
    debug1: Next authentication method: publickey
    debug1: Offering RSA public key: /public/home/XiaAnRen/.ssh/id_rsa
```

`top`中的`load average`非常高，因此检查占用最高的任务，看到是和NFS相关的内核线程：
```bash
>>> ps aux --sort=-%cpu | head -20
    USER PID   %CPU %MEM VSZ RSS TTY STAT START TIME   COMMAND
    root 36978 23.9 0.0  0   0   ?   D    Jan14 327:00 [172.16.2.251-ma]
    root 35075 12.2 0.0  0   0   ?   R<   Jan14 168:14 [kworker/2:0H]
```

同时检查日志信息有：
```bash
>>> cat /var/log/messages | grep nfs | tail -10
    Jan 14 09:34:03 node15 kernel: nfs4_reclaim_open_state: 5 callbacks suppressed
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:03 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
    Jan 14 09:34:06 node15 kernel: NFS: nfs4_reclaim_open_state: Lock reclaim failed!
```

因此推测是NFS挂载的问题。

在node15进一步检查挂载情况：
```bash
>>> mount | grep /public
    /etc/auto.direct on /public type autofs (rw,relatime,fd=18,pgrp=2102,timeout=300,minproto=5,maxproto=5,direct,pipe_ino=50523)
    mgt-ib:/public on /public type nfs (rw,relatime,sync,vers=3,rsize=1048576,wsize=1048576,namlen=255,hard,proto=tcp,timeo=600,retrans=2,sec=sys,mountaddr=172.16.2.251,mountvers=3,mountport=20048,mountproto=udp,local_lock=none,addr=172.16.2.251)
    mgt-ib:/public on /public type nfs4 (rw,relatime,vers=4.1,rsize=1048576,wsize=1048576,namlen=255,hard,proto=tcp,port=0,timeo=600,retrans=2,sec=sys,clientaddr=172.16.2.17,local_lock=none,addr=172.16.2.251)
```

而其他没有问题的计算节点都是：
```bash
>>> mount | grep /public
    /etc/auto.direct on /public type autofs (rw,relatime,fd=18,pgrp=2003,timeout=300,minproto=5,maxproto=5,direct,pipe_ino=17584)
    mgt-ib:/public on /public type nfs (rw,relatime,sync,vers=3,rsize=1048576,wsize=1048576,namlen=255,hard,proto=tcp,timeo=600,retrans=2,sec=sys,mountaddr=172.16.2.251,mountvers=3,mountport=20048,mountproto=udp,local_lock=none,addr=172.16.2.251)
```

因此问题的根源在于node15出现了重复挂载。

### 解决方案

指定文件系统类型为NFS v4，并采用懒卸载机制防止因被占用而导致卸载失败：
```bash
umount -lt nfs4 /public
```
