## 20260325-配置节点X11图形界面转发

### 问题描述

只有直接登录节点才能成功打开X11图形窗口。

### 解决方案

经测试必须使用`ssh -Y node`才能成功（`ssh -X node`会失败）。

因此修改`~/.ssh/config`实现持久化（`node*`等计算节点由于缺少`xauth`所以被排除在外）：
```ssh-config
Host mgt login2 gpu01
  User USER
  ForwardX11 yes
  ForwardX11Trusted yes
```

### 问题描述

`xclock`成功。

使用`matplotlib`会出现警告：
```python
import matplotlib.pyplot as plt
_ = plt.figure()
```

### 解决方案

改用`VcXsrv`而非`Xming`。

下载地址：<https://sourceforge.net/projects/vcxsrv>

安装完成后启动`xlaunch.exe`，全程保持默认配置，中间`Save configuration`得到`config.xlaunch`，再以后就只需要运行`config.xlaunch`。

### 问题描述

`matplotlib`的警告消失。

如果登录期间运行过相关的脚本，则在退出登录的时候会卡死，需要额外执行一次`Ctrl+C`。

### 解决方案

在`~/.bashrc`中添加：
```bash
[[ -n "$SSH_CLIENT" && -n "$DISPLAY" && -z "$DBUS_SESSION_BUS_ADDRESS" ]] && eval "$(dbus-launch --sh-syntax)"
```

在`~/.bash_logout`中添加：
```bash
[[ -n "$SSH_CLIENT" && -n "$DISPLAY" && -n "$DBUS_SESSION_BUS_PID" ]] && \
  ps -p "$DBUS_SESSION_BUS_PID" -o comm= | grep -qx dbus-daemon && kill -TERM "$DBUS_SESSION_BUS_PID"
```
